Mac OS(Maverick):生产力提高的一剂良药

很可惜，由于资金问题，现在还没有购买Macbook的计划。但是作为纯屌丝，在我国内读研准备GRE考试的时候，就是靠着以后去美国买全套苹果设备的信念度过的那段难熬的日子的。那时，也用我的Acer 4930装过Mac OS Snow leopard黑苹果，一直在意淫，从未被超越。当时只是为了体验新鲜和折腾，幻想着手中的这台笔记本是我的第一台苹果设备，仅此而已。

四年过去了，iPod、iPad、iPhone已经被我玩腻了，换回了尚可折腾的搭载Android的Nexus 4。但这些还都是玩物，iPad并不是个称职的文献阅读器，主要取决于使用者的定力。后来，随着自己开始学R，敲代码的时间越来越多，又接触了markdown和github，有意无意地逐渐离自己幻想中的工作状态更接近了，就是用计算机去解决一些普通生物、化学实验解决不了的问题，找寻同专业或同方向中一定的差异化。

最近由于天气等不可抗力原因，实验无法有进展，于是自己在做文献计量方面的研究，发现可以很好的将计算机的文本挖掘和具体的科研方向相结合，就潜心研究了下去。推荐大家看一个期刊Scientometrics，就是文献计量方面的一个sci期刊。我也希望自己能把这些结果整理一下发在上面。

前面引子写的有点长，似乎还没点到题。我之所以又想折腾黑苹果系统，是因为我现在用的电脑是实验室给配的，平时是用个人账号登陆的，没有管理员权限，但导师给了我一个通用的管理员账号，允许我安装软件。但每次我需要装软件的时候，都要切换用户，有时有些软件，比如python的IDE在用非管理员账号安装使用的时候，会遇到很多无权限的问题，非常恼人。于是，我就想我装个虚拟机，一劳永逸，我所有的工作都在虚拟机里，等我毕业了，直接把虚拟机拷贝走，放在其他电脑上还可以继续用。于是，我就安装了vmware，也可以安装virtualbox。因为我这台电脑配置挺高，i7的CPU，8G内存，我才考虑干脆用mac os吧，于是的于是就。

如何安装vmware虚拟机的Mac OS

网上有很多教程，有老外写的，有中文的博客上的，有pcbeta里的帖子。我这就不赘述了，主要总结一下思路：

  1. 确认你的电脑内存是否足够，如果想用mac虚拟机作为你的生产力工具，分配给虚拟机的内存不能少于4G，硬盘空间不少于40G
  2. 下载vmware 10和maverick的懒人版（文件后缀为.cdr，是网友用原版镜像制作的专门用于黑苹果安装的镜像文件），一定要用懒人版，原版的如果不替换一些文件是不可以安装的。
  3. 下载vmtools.iso即可，如果你的屏幕是超清的（分辨率高于1080p），还需要下载vsvga的驱动，这些都在pcbeta的那个帖子里能找到。
  4. 开始安装吧。

关于软件，推荐大家买正版，如果像我一样囊中羞涩，可以去macx.cn和macappbox.com下载mac的软件。

mac神器alfred

很多人已经奔跑着歌颂过这个软件了，我在此毫不掩饰的表达我对这个软件的喜爱，可能就因为她，我会加快我购入macbook的计划。

alfred是干嘛的？

官网在此。她是一款可以自由设置工作流的软件，会把多步骤的电脑操作简化为简单的命令，一步到位。我的描述可能并不准确，建议大家去看这个知乎的回答。

一个方便hexo用户的alfred workflow

我的这个博客就是用hexo解析到github page上的。每次想写博客的时候，要先在terminal里面cd到hexo的目录，创建，再打开这个.md文件。用alfred就可以简化成一步。

首先双击terminal那个方块，开始编辑，

将cd 后的文件路径改成自己的即可，

然后即可用快捷键呼出alfred，然后输入hexo new mynewblog，其中mynewblog就是你要写的博客题目，然后就会自动用系统默认的编辑器打开这个.md文件。

hexo workflow可以在这里下载，点击。

其实，更推荐大家自己设置好了，很简单很简单。

短暂的结束语

其实，Mac的优点有很多，吸引人的地方也因人而异。阳志平的博客中也写了个初学者介绍教程，推荐想了解mac的童鞋去看看。

我的这篇文章是作为我用Mac的引子，一个起点，我已经把R、python、博客等都搬到Mac了，我的windows 7上的软件都被我删光了，为了腾出空间给这个mac虚拟机，还好电脑配置高，用这个虚拟机很流畅。

不过，的确不推荐普通用户用mac，只推荐会经常用到键盘、敲代码的童鞋，相信很多码农的终极目标都是mac吧，linux之流是给牛人用的。

没想到我一个做生态做实验的，能在马年如此惬意的敲代码，希望自己马到成功吧，实验快快进展起来。
